# v0.1 Bugs Found and Fixes

**Date:** 2026-04-05

## Bug 1: Gemini API 429 ‚Äî Quota exceeded

**Symptom:** App loaded fine, but logging a meal returned a Gemini API 429 error.

**Root cause:** The code used model name `gemini-2.0-flash` which was not available on the user's API key. The working model endpoint was `gemini-flash-latest`.

**Fix:** Changed `GEMINI_API_URL` in `src/ai/gemini.ts` from `gemini-2.0-flash` to `gemini-flash-latest`.

**Lesson:** Test the API key with curl first before assuming the model name. Different Gemini API keys may have access to different model names.

---

## Bug 2: SQLite NativeDatabase.prepareAsync NullPointerException

**Symptom:** After logging a few meals, a `NativeDatabase.prepareAsync has been rejected` error appeared with a `java.lang.NullPointerException`.

**Root cause:** Two issues:
1. `expo-sqlite` version mismatch ‚Äî installed v55 but Expo SDK 54 expected v16. Fixed with `npx expo install expo-sqlite` which picks the compatible version.
2. Race condition ‚Äî multiple calls to `getDatabase()` could trigger parallel `openDatabaseAsync` calls before the first one resolved.

**Fix:**
- Used `npx expo install` instead of `npm install` for Expo-managed packages
- Changed database singleton from a plain variable to a **promise singleton** pattern ‚Äî ensures only one init ever runs
- Passed parameters as an array to `runAsync` instead of spread args

**Lesson:** Always use `npx expo install` for Expo packages ‚Äî it handles version compatibility. For async singletons, cache the promise, not the result.

---

## Bug 3: React hooks order violation

**Symptom:** Error: "Rendered more hooks than during the previous render" ‚Äî React detected a change in hook call order.

**Root cause:** Added a `useCallback` for `renderMeal` after the early return (`if (!apiKeySet) return ...`). When `apiKeySet` was false, the hook didn't run; when true, it did ‚Äî changing the hook count between renders.

**Fix:** Moved the `useCallback` before the early return so all hooks are always called in the same order regardless of which branch renders.

**Lesson:** In React, all hooks must be called before any early returns. This is a fundamental React rule ‚Äî hooks can't be conditional.

---

## Bug 4: Meal list flickering on new entry

**Symptom:** When a new meal is logged, all existing meal cards in the list flicker/re-render.

**Root cause:** `FlatList` re-renders all items when the `meals` state array is replaced. The `renderMeal` function was recreated on every render.

**Partial fix:** Extracted `MealCard` as a `memo`ized component and wrapped `renderMeal` in `useCallback`. Reduced but didn't fully eliminate flickering.

**Status:** Deferred to v0.3 when the dashboard UI is rebuilt.
