- `App.tsx` — added screen state and navigation between home/settings

## Bugs found and fixed
- **Goals not persisting**: `getDailyGoals` query with `ORDER BY effective_from DESC` didn't distinguish rows with the same date — fixed by adding `id DESC` tiebreaker
- **API key shown for all providers**: single `api_key` setting shared across providers — changed to per-provider keys (`api_key_gemini`, etc.)
- **Masked key shown on re-entry**: saved key loaded back as password dots — replaced with "Key saved" status + "Change" link
- **Save blocked by unsupported providers**: Claude/OpenAI chips removed entirely until implemented

## Design decisions
- API keys stored per-provider, ready for future BYOM expansion
- Removed Claude/OpenAI provider chips — only Gemini shown until others are wired up
- "Key saved" + "Change" pattern instead of loading masked key back into input
- Simple state-based navigation — no navigation library needed yet
