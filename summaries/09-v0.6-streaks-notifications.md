# Session 09: v0.6 Build — Streaks + notifications

**Date:** 2026-04-08

## What was done
- Added streak tracking — counts consecutive days with logged meals, shown as 🔥 badge
- Added local push notifications — 3 daily reminders (8am, 1pm, 7pm) with fun randomized messages
- Added notifications toggle in Settings with permission handling
- Built first standalone APK via EAS Build (v0.5), established APK build as part of workflow
- Added EAS build step to WORKFLOW.md — build APK after every version merge

## Files created/changed
- `src/db/database.ts` — added `getCurrentStreak()` query
- `src/screens/HomeScreen.tsx` — streak badge in title row, streak in data state
- `src/notifications.ts` — new file: notification scheduling, permission handling, toggle
- `src/screens/SettingsScreen.tsx` — notifications toggle with Switch component
- `app.json` — added Android package name, permissions, EAS project ID
- `eas.json` — new file: EAS build config for preview (APK) and production

## Bugs found and fixed (from v0.5 testing)
- **expo-file-system EncodingType.Base64 undefined**: SDK 54 runtime issue — used string literal `'base64'`
- **expo-file-system deprecated API**: SDK 54 moved to new File/Directory API — imported from `expo-file-system/legacy`
- **Voice recording stale closure**: `useState` for recording instance was stale in `stopRecording` — switched to `useRef`

## Design decisions
- Streak derived from meals table (no new table) — counts consecutive days backwards from today
- Notifications use expo-notifications with `DAILY` trigger type
- 5 randomized reminder messages for variety
- Notifications only work in standalone APK, not Expo Go (SDK 53+ limitation)
- Streak badge: orange pill with fire emoji, only shown when streak > 0
