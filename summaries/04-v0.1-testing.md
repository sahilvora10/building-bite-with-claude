**Date:** 2026-04-05

## What was done
- Started Expo dev server with tunnel mode (needed for WSL2 → phone connectivity)

## Bugs found and fixed
1. **Gemini 429** — wrong model name in API URL → changed to `gemini-flash-latest`
2. **SQLite NullPointerException** — version mismatch + race condition → `npx expo install` + promise singleton pattern
3. **React hooks order** — `useCallback` after early return → moved before early return
4. **Flickering meal list** (deferred) — memoized `MealCard` but didn't fully resolve, will fix in v0.3 with new UI

## Process learnings
- **WSL2 networking quirk:** Expo's default LAN mode doesn't work from WSL2 to phone. Must use `--tunnel` mode.
- **Always use `npx expo install`** for Expo packages, not `npm install` — handles version compatibility automatically.
- **Claude can't play video** — screenshots (jpg/png) work, video files don't. Use screenshots for bug reports.
- **curl test first** — testing the API key with curl before app integration would have caught the model name issue faster.
- **Feedback file workflow:** phone → `\\wsl$\Ubuntu\home\...\journey\feedback\` → Claude reads images.
