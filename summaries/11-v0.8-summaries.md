# Session 11: v0.8 Build — Summaries + non-food rejection

## What was done
- Built SummariesScreen with 7-day average card and daily macro breakdown
- Progress bars for each macro relative to daily goals
- Expandable day cards — tap to see individual meals logged that day
- Added non-food rejection: low confidence or 0-calorie results show fun alert instead of logging
- Merged v0.6 (streaks), v0.7 (recipes), v0.8 (summaries) to master with APK builds

## Files created/changed
- `src/screens/SummariesScreen.tsx` — new summaries screen with weekly avg + daily breakdown

- `src/screens/HomeScreen.tsx` — added 📊 icon, non-food rejection guard, onOpenSummaries prop
- `App.tsx` — added summaries screen navigation

## Bugs found and fixed
- **Non-food images logged**: Photos of non-food items created 0-calorie cards or hallucinated food (mic → cola). Fixed with confidence + calorie check before insert.
- **Recipe only analyzed first photo**: Multi-photo recipes only sent photos[0] to Gemini. Fixed with imagesBase64 array support.

## Design decisions
- Weekly average shows over/under goal comparison
- Daily breakdown uses color-coded progress bars matching ring colors
- Day cards are tappable — expand to show meal name, time, and macros
- Non-food threshold: confidence < 0.3 OR calories === 0
- Four randomized fun rejection messages
